Introduction to Agents

Specialized AI assistants with isolated contexts and custom capabilities

What Are Agents?

Agents (or subagents) are specialized AI assistants that run in their own isolated context window with:

  • Custom system prompts - Focused instructions
  • Independent permissions - Controlled tool access
  • Separate context - Won't clutter main conversation
  • Configurable models - Use Haiku for speed, Opus for quality

Key Benefits

  • Context Preservation - Keep exploration separate from implementation
  • Tool Restrictions - Enforce read-only for research agents
  • Cost Control - Route to cheaper/faster models
  • Parallelization - Run multiple agents simultaneously
  • Reusability - Share agents across projects
⚠ Important Limitation

Agents cannot spawn other agents (no nesting). For multi-level delegation, chain agents from the main conversation.

💡 Pro Tip

The Explore Agent

Your go-to for fast, safe codebase research

When to Use

  • Finding files by patterns or content
  • Understanding code structure
  • Searching for implementations
  • Gathering context before changes
  • Answering questions about the codebase

Thoroughness Levels

  • quick - Basic searches, known locations
  • medium - Moderate exploration
  • very thorough - Comprehensive analysis

Example Prompt

# Ask Claude to use the Explore agent: "Find all files that handle user authentication. Be very thorough." # Claude will spawn an Explore agent # with read-only tools (Glob, Grep, Read) # running on the fast Haiku model
✅ Best Practice

Use Explore instead of direct Grep/Glob for open-ended searches - it handles multiple iterations automatically.

Creating Custom Agents

Define specialized agents for your workflow

Storage Locations

~/.claude/agents/ # User-level (all projects) code-reviewer.md debugger.md .claude/agents/ # Project-level (shared via git) api-specialist.md test-writer.md

Priority Order

  1. Session flag (--agents)
  2. Project (.claude/agents/)
  3. User (~/.claude/agents/)
  4. Plugin agents

Creation Methods

1. Interactive CLI

/agents # Opens agent management UI

2. Manual File Creation

Create .md files in the agents directory with YAML frontmatter.

3. CLI Flag (Session-only)

claude --agents '{ "my-agent": { "description": "...", "prompt": "...", "tools": ["Read", "Grep"] } }'

Agent Configuration Example

A complete custom agent definition

--- name: code-reviewer description: Expert code review specialist for security and best practices tools: - Read - Grep - Glob - Bash model: sonnet permissionMode: default --- # Code Review Specialist You are an expert code reviewer focusing on: - Security vulnerabilities - Performance issues - Code maintainability - Best practices adherence Always provide specific line references and actionable feedback.

Key Components

Frontmatter (YAML)

  • name - Unique identifier
  • description - When to use (helps Claude decide)
  • tools - Allowed tools (restrictive)
  • model - AI model to use

Body (Markdown)

  • Custom system prompt
  • Behavioral instructions
  • Domain expertise
  • Output format requirements
💡 Tip

Include "use proactively" in the description to encourage Claude to delegate automatically.

Agent Frontmatter Reference

Complete configuration options for custom agents

Field Required Description
name Yes Unique identifier for the agent
description Yes When to use this agent (helps Claude auto-delegate)
tools No List of allowed tools (e.g., Read, Grep, Edit)
disallowedTools No Tools to explicitly deny from inherited set
model No sonnet | opus | haiku | inherit
permissionMode No default | acceptEdits | dontAsk | plan
skills No Skills to preload into agent context
hooks No Lifecycle hooks: PreToolUse, PostToolUse, Stop
⚠ Permission Modes

bypassPermissions skips all safety checks - use with extreme caution. Prefer acceptEdits for automation.

Introduction to Skills

Reusable prompts and workflows Claude loads when relevant

What Are Skills?

Skills are folders of instructions, scripts, and resources that teach Claude how to complete specific tasks in a repeatable way.

Key Characteristics

  • Minimum requirement: A SKILL.md file
  • Can include: Templates, examples, scripts
  • Progressive loading: Context-efficient
  • Hot-reloading: Changes apply immediately

Invocation Methods

① Automatic

Claude detects when a skill is relevant and loads it automatically based on the description.

② Manual (Slash Command)

/skill-name /skill-name argument1 argument2
🔄 Skills vs Agents

Skills run in the main conversation context by default. Agents run in isolated contexts. Use context: fork to run a skill in a subagent.

SKILL.md Structure

How to organize a skill

Directory Layout

~/.claude/skills/ my-skill/ SKILL.md # Required: main instructions template.md # Optional: fill-in template examples/ sample.md # Example outputs scripts/ validate.sh # Helper scripts

Storage Locations

  • ~/.claude/skills/ - Personal (all projects)
  • .claude/skills/ - Project (shared via git)
  • Plugin skills directory

SKILL.md Example

--- name: pr-summary description: Generate PR summaries from git diff argument-hint: [branch-name] --- # PR Summary Generator Generate a concise PR summary based on the changes in $ARGUMENTS. ## Steps 1. Get the diff: !`git diff main...$ARGUMENTS` 2. Analyze changes by category 3. Write summary with: - Title (under 70 chars) - Bullet points of changes - Testing notes

Skill Frontmatter Reference

Complete configuration options for skills

Field Purpose
name Display name and slash command name (lowercase, max 64 chars)
description What the skill does - helps Claude trigger automatically
argument-hint Hint for expected arguments (e.g., [file] [format])
disable-model-invocation Set true to prevent auto-triggering (manual only)
user-invocable Set false to hide from / menu (background knowledge)
allowed-tools Tools Claude can use without permission when skill is active
model Specific model to use when skill executes
context Set to fork to run in isolated subagent
agent Subagent type to use with context: fork
hooks PreToolUse, PostToolUse, Stop hooks scoped to skill

Built-in vs Custom Skills

Understanding skill types and priority

Built-in Slash Commands

System commands (not customizable skills):

/clear # Clear conversation /compact # Compress context /config # Open settings /context # Visualize usage /help # Get help /memory # Edit CLAUDE.md /model # Select model /plan # Enter plan mode /resume # Resume session /rewind # Undo changes

Custom Skill Locations

  • Enterprise Managed settings (highest)
  • Personal ~/.claude/skills/
  • Project .claude/skills/
  • Plugin Plugin skills directory
🔄 Priority Rule

Higher-priority locations override lower ones. If a skill and command share the same name, the skill takes precedence.

Pre-built Skills (Anthropic)

  • PDF - Creation, extraction, form filling
  • Word (DOCX) - Document editing
  • PowerPoint (PPTX) - Presentations
  • Excel (XLSX) - Spreadsheets

Slash Command Patterns

Invocation syntax and dynamic context injection

Basic Invocation

# Simple invocation /skill-name # With arguments /fix-issue 123 /deploy production # Multiple arguments /convert file.md pdf

Argument Variables

# In SKILL.md: $ARGUMENTS # All arguments $0 # First argument $1 # Second argument $ARGUMENTS[0] # Same as $0

Dynamic Context Injection

Execute shell commands and inject output:

--- name: pr-review --- # PR Review ## Current Changes !`git diff main...HEAD` ## PR Comments !`gh pr view --comments` Review the above and provide feedback.
💡 Backtick Syntax

Use !`command` to run a shell command and replace the placeholder with its output before sending to Claude.

Decision Matrix

Use Case Best Choice
Quick, repeatable task Skill
Needs isolated context Agent
Verbose output isolation Agent
Tool restrictions needed Agent
Parallel execution Multiple Agents
Team-shared workflows Skill in .claude/
Skill + isolation Skill + context: fork

Best Practices Summary

Key recommendations for agents and skills

🔧 Agents

  • Design focused, single-purpose agents
    One agent = one specialized task
  • Write detailed descriptions
    Helps Claude know when to delegate
  • Limit tool access
    Only include tools the agent needs
  • Use Haiku for exploration
    Fast and cost-effective for read-only tasks
  • Check agents into version control
    Share team workflows via .claude/agents/
  • Use parallelization
    Spawn multiple agents for independent research

⚡ Skills

  • Keep SKILL.md concise
    Under 500 lines; use supporting files for references
  • Use dynamic context injection
    !`command` for fresh data
  • Control invocation appropriately
    Use disable-model-invocation for sensitive skills
  • Test both invocation methods
    Verify auto and manual (/name) work correctly
  • Use context: fork wisely
    For tasks producing verbose output
  • Provide argument hints
    Use argument-hint for discoverability
🎯 Golden Rule

Context is your most valuable resource. Use agents to isolate verbose operations and skills to encode repeatable workflows.
